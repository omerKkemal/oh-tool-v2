Introduction

This document provides comprehensive documentation for the SpecterPanel API, including both machine-facing endpoints (used by remote agents) and web-facing endpoints (used by the dashboard interface).

Authentication

All API requests require authentication via an API token. There are two ways to provide the token:

  • GET requests: Include as a query parameter (?token=ABC123)
  • POST requests: Include in the JSON request body ({"token":"ABC123"})

Base URL

All examples use SERVER as a placeholder for your server's base URL (e.g., https://panel.example.com). Replace this with your actual server address when making requests.

Rate Limiting

The API enforces rate limits to prevent abuse. If you exceed the limit, you'll receive a 429 Too Many Requests response. Wait before making additional requests.

/api/ApiCommand/<target_name>

Retrieve pending commands for a specific target. Remote agents should periodically poll this endpoint to check for new commands.

Note: Agents should include their IP and OS information when available to help with command targeting.

Parameters

Name Type Location Required Description
tokenstringquery*API token for authentication
ipstringqueryAgent's IP address (optional but recommended)
osstringqueryAgent's operating system (optional but recommended)
target_namestringpath*Unique identifier for the target machine

Responses

200 Success

{
  "allCommand": [
    ["ID", "email", "target_name", "command", "condition", "update"],
    ["cmd123", "admin@example.com", "demo_target", "whoami", "linux", "2023-08-11 12:00:00"],
    ...
  ]
}

403 Invalid token

{ "Error": "Invalid token" }
// JavaScript (fetch)
fetch('http://SERVER/api/ApiCommand/demo_target?token=ABC123&ip=10.0.0.2&os=linux')
  .then(response => {
    if (!response.ok) throw new Error(`HTTP error! status: ${response.status}`);
    return response.json();
  })
  .then(data => console.log('Received commands:', data))
  .catch(error => console.error('Error fetching commands:', error));
# Python (requests)
import requests
params = {'token': 'ABC123', 'ip': '10.0.0.2', 'os': 'linux'}
try:
    response = requests.get('http://SERVER/api/ApiCommand/demo_target', params=params)
    response.raise_for_status()
    print("Commands:", response.json())
except requests.exceptions.RequestException as e:
    print("Error fetching commands:", e)
# cURL
curl -X GET "http://SERVER/api/ApiCommand/demo_target?token=ABC123&ip=10.0.0.2&os=linux" -H "Accept: application/json"

/api/Apicommand/save_output

Submit command execution results back to the server. Agents should call this after executing received commands.

Note: The output should be an array of arrays, each containing a command ID and its corresponding output.

Request Body (JSON)

NameTypeRequiredDescription
tokenstring*API token matching the registered target
target_namestring*Target identifier that executed the commands
outputarray*Array of [command_id, output_text] pairs
ipstringAgent's current IP address (optional)
osstringAgent's operating system (optional)

Responses

200 Success

{ "message": "Outputs were saved successfully" }

403 Invalid token

{ "Error": "Invalid token or target" }
// JavaScript (fetch)
const commandOutputs = [["cmd123", "user123\n"], ["cmd124", "Linux hostname 5.4.0-104-generic\n"]];
fetch('http://SERVER/api/Apicommand/save_output', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ token: 'ABC123', target_name: 'demo_target', ip: '10.0.0.2', os: 'linux', output: commandOutputs })
}).then(response => response.json()).then(data => console.log('Server response:', data)).catch(error => console.error('Error:', error));
# Python (requests)
import requests
output_data = {'token': 'ABC123', 'target_name': 'demo_target', 'ip': '10.0.0.2', 'os': 'linux', 'output': [['cmd123', 'user123\n'], ['cmd124', 'Linux hostname 5.4.0-104-generic\n']]}
try:
    response = requests.post('http://SERVER/api/Apicommand/save_output', json=output_data)
    print("Server response:", response.json())
except Exception as e: print("Error:", e)
# cURL
curl -X POST "http://SERVER/api/Apicommand/save_output" -H "Content-Type: application/json" -d '{"token":"ABC123","target_name":"demo_target","ip":"10.0.0.2","os":"linux","output":[["cmd123","user123\\n"],["cmd124","Linux hostname 5.4.0-104-generic\\n"]]}'

Web interface endpoint for managing commands. GET requests render the web interface, while POST requests add new commands for the specified target.

Note: Requires valid session authentication (web login).

Parameters (POST JSON)

NameTypeRequiredDescription
inputstring*Command text to be executed by the target

Responses

200 POST success

// JavaScript (fetch) - requires session cookie
fetch('/api_command/demo_target', {method:'POST', headers:{'Content-Type':'application/json'}, credentials:'include', body:JSON.stringify({input:'whoami'})})
.then(r=>r.json()).then(d=>console.log('Command saved:',d)).catch(e=>console.error(e));
# Python (requests) with session
import requests
session = requests.Session()
# login first...
try:
    r = session.post('http://SERVER/api_command/demo_target', json={'input':'whoami'})
    print(r.json())
except Exception as e: print(e)
# cURL with cookie
curl -b cookies.txt -X POST "http://SERVER/api_command/demo_target" -H "Content-Type: application/json" -d '{"input":"whoami"}'

/api_link

Manage API links. GET requests render the web interface, while POST requests create new API links.

Note: Requires valid session authentication. POST expects form data.

Parameters (POST Form)

NameTypeRequiredDescription
target_namestring*Target associated with the API link
linkstring*URL or endpoint for the API link
action_typestring*HTTP method (GET, POST, etc.)
conditionintegerCondition for the link (optional)
 
// JavaScript (fetch) - Form POST
const fd = new FormData();
fd.append('target_name','demo_target'); fd.append('link','http://example.com/api'); fd.append('action_type','GET'); fd.append('condition','1');
fetch('/api_link', {method:'POST', credentials:'include', body:fd})
.then(()=>console.log('API link created')).catch(e=>console.error(e));
# Python (requests) with session
form = {'target_name':'demo_target','link':'http://example.com/api','action_type':'GET','condition':'1'}
try:
    r = session.post('http://SERVER/api_link', data=form)
    print("API link created")
except Exception as e: print(e)
# cURL
curl -b cookies.txt -X POST "http://SERVER/api_link" -F "target_name=demo_target" -F "link=http://example.com/api" -F "action_type=GET" -F "condition=1"
